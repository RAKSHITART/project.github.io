2. Customer Navigation System

Purpose: This system helps guide customers along the shortest path between two points in the mall.

Implementation Details:

  • The mall is represented as a graph, where nodes represent locations (e.g., shops) and edges represent paths between them.
  • Dijkstra's Algorithm calculates the shortest path between the starting and destination points.
  • A Binary Search Tree (BST) stores the shop locations, enabling fast and efficient lookups for customer destinations.

3. Searching for Products in a Mall Directory

Purpose: This system allows customers to search for specific product names or categories (e.g., "electronics," "fashion") within the mall's product directory to quickly find relevant items or stores.

Implementation Details:

  • The Rabin-Karp Algorithm is used to search for products or categories efficiently by comparing hash values.
  • The algorithm works by calculating a hash value for the search term and matching it against hash values of substrings in the product directory.
